Software applications provide a variety of mechanisms for logging everything that can happen within a system. The analysis of these log files in order to extract some knowledge about security issues is a real challenge. In this paper, we propose a methodology for analyzing log files based on the development of decision support system. The goal of this methodology is to collect log file events from different machines and save the necessary data to a data warehouse in order to extract security knowledge, monitoring, reporting, get operational information, and make better decisions. As illustration, we have considered a concrete case study on web server log files based on data warehousing techniques to diagnose the state of the system security, to track key performance indicators, and to identify abnormal behavior and detect some malicious events within computer systems in the hope of taking counter-measures against them. The obtained results are promising; we can easily visualize servers health and monitor problems.
